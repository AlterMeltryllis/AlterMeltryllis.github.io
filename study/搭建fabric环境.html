# 搭建Fabric环境

# 0x00 开始

虽然很早前就说要开始做毕业设计但是一直没啥进展，最近和学长交流了后决定还是先开始试试（绝对不是时间不够了🥲）最后决定基于Forbic搭建一个区块链的应用程序。因为在搭建的过程中发现是有官方的文档的，基本上按照文档就可以完成搭建。

环境搭建参考：fabric官方文档汉化 (opens new window) CSDN的部分文章

因为本机上有已经安装好的其他内容，所以此处没有详细展示除开下文展示的内容还需要安装git和cURL，这俩的下载可以看我说的官方文档有写。

# 环境配置

先输入go env出现以下内容，此时我们需要看的内容为GOBIN、GOPATH和GOROOT，其中GOPATH为工程项目的目录；GOBIN是GOPATH下的bin目录；GOROOT是GO语言安装地址。

然后需要进行配置环境变量：

1.创建一个文件夹作为工程文件夹（此处创建在终端打开的默认文件夹或者直接在终端输入CD $HOME可到达）

mkdir go
1

2.编辑.bash_profile文件，没有就新建一个（mac查看隐藏文件快捷键：command + Shift + .）然后在文件中添加以下内容：

#GOPATH
export GOPATH=$HOME/go

#GOBIN
export GOBIN=$GOPATH/bin
export PATH=$PATH:$GOBIN
1
2
3
4
5
6

保存后执行source命令：

source .bash_profile
1

3.使用go env检查出现以下结果发现已写入内容：

# 0x02 安装测试实例以及Docker镜像

首先进入Go的工作空间，官方给的方法是使用以下命令：

curl -sSL https://bit.ly/2ysbOFE | bash -s
1

此处安装的为最新版本的内容，如果你想安装指定版本需要使用别的命令，因为我没有进行这个内容，建议参考官方文档。

我在使用curl命令的时候会出现以下问题：

 l15hee7@L15hee7deMBP project % curl -sSL https://bit.ly/2ysbOFE | bash -s -- -h
curl: (35) OpenSSL SSL_connect: SSL_ERROR_SYSCALL in connection to bit.ly:443
1
2

我的解决方案是直接访问以下网站，因为上面的命令我觉得是下载并执行bootstrap.sh文件，因此下载下来本地执行也是一样的。

https://raw.githubusercontent.com/hyperledger/fabric/master/scripts/bootstrap.sh

在本地新建一个sh文件并将网址上的内容复制到里面，输入./bootstrap.sh执行文件。在执行过程中会出现以下问题：

Clone hyperledger/fabric-samples repo

===> Cloning hyperledger/fabric-samples repo
正克隆到 'fabric-samples'...
致命错误：无法访问 'https://github.com/hyperledger/fabric-samples.git/'：LibreSSL SSL_read: error:02FFF03C:system library:func(4095):Operation timed out, errno 60
fabric-samples v2.4.9 does not exist, defaulting to main. fabric-samples main branch is intended to work with recent versions of fabric.
致命错误：不是 git 仓库（或者任何父目录）：.git
1
2
3
4
5
6
7

你会发现在下载测试样本时会有这个问题，我理解的是下载的版本太高了，这个地方还没有分类。解决方法是直接在文件夹中使用以下命令：

git clone https://github.com/hyperledger/fabric-samples.git
1

输入git tag查看已经有的版本，然后使用git checkout v2.4.3检出最新版本即可

此时文件夹中会有以下内容：

(base) l15hee7@L15hee7deMacBook-Pro project % ls
bin		bootstrap.sh	builders	config		fabric-samples
1
2

然后将bin和config文件夹复制到fabric-samples文件夹后就可以进行测试了。

cd fabric-samples/test-network

./network.sh up
