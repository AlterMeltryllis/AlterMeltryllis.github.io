# 解决git@github.com: Permission denied (publickey)问题

# 0x00 问题出现

因为某些原因电脑系统被重装了，但是有把之前生成的git密钥保存下来准备通过写config文件的方法继续使用之前的密钥访问git，本来一开始是没什么问题的，但是在经过了两三周在github上提交代码的时候出现了以下错误：

git@github.com's password:
Permission denied, please try again.
1
2

它是有让我输入密码的，所以我尝试了自己生成密钥时的密码和github密码（讲道理我觉得应该就是生成密钥时的密码，但是网上有人说是github密码）但是这俩都是显示的错误。然后尝试了删除密钥重新生成等各种方法最终无解。

# 0x01 解决方案

在查询资料的时候找到了github的官方文档？ (opens new window)然后里面提到了这个问题：

很庆幸的是我测试完的端口是可行的，但是找到这个解决方法的过程确实漫长。

然后需要我们在config文件（~/.ssh/config）中编写相关信息：

Host github.com
    HostName ssh.github.com
    IdentityFile ~/.ssh/id_rsa.github
    User git
    Port 443
1
2
3
4
5

注意这个地方是在HostName后面的内容上加ssh，然后使用ssh -T git@github.com对连接进行测试即可。
